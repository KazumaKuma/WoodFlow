# WoodFlow Documentation Standard

## Purpose

This document defines the documentation conventions used throughout the WoodFlow project.

The goal is to keep all project documentation consistent, readable and maintainable from the beginning of development.

## Standard Document Header

Every Markdown document within the project should begin with the following metadata block:

```yaml
---
Project: WoodFlow
Document:
Version: v0.1.0
Status: Active
Author: Kazuma Kuma
Created: 2026-07-01
Last Updated: 2026-07-01
---
```
